About

This app is a QR code reader with additional features. These features are meant to protect users from potentially malicious QR codes. You can enable and disable these security checks from the Settings-menu. Note that these security checks (including Google Safe Browsing) are not perfect. We do our best to keep false positives at a minimum, but they can still happen. If you encounter such a false positive, please let us know so we can improve our App.

In case you want to generate your own signed QR code or find out more about the Secure QR project, head on over to our website.

Security Checks
Error Rate

The app checks for an abnormally high error rate in the QR code. A high error rate might indicate that the code has been tampered with.

Phishtank

URLs are verified against the Phishtank API to detect malicious URLs.

Redirect Count

The app warns if a scanned QR code wants to redirect the user a large number of times, as this might indicate that the URL is trying to hide its "real" identity or create a redirection loop.

Google Safe Browsing

All URLs are checked against Google's Safe Browsing lists of known malicious URLs.

Dynamic DNS

The user is warned if an URL uses a hostname that is backed by a dynamic DNS service. This is due to the fact, that these hostnames have a very high abuse potential due to their volatility.

Robots.txt

The app checks whether the site may be indexed by search engines. If a QR code is scanned in public, and the site disallows all crawlers, this quite strange. If you put up a QR code in a public place, you want your website to be found.

Note: this check is disabled by default due to a high possibility of false positives.

Website Age

The Wayback Machine is asked about when it has first seen the root page of the scanned website. Phishing sites are created and destroyed quite rapidly. This is why "young" websites, whose QR codes are found in public places, might be malicious.

Note: this check is disabled by default due to a high possibility of false positives.
